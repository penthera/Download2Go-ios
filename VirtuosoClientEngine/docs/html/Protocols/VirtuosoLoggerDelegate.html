VirtuosoLoggerDelegate Protocol Reference

Conforms to NSObject
Declared in VirtuosoLogger.h

Overview

For those who wish to use their own logging system or report metrics to non-Virtuoso locations/trackers. Virtuoso will send all log events to the registered delegate if the VirtuosoLogger would have otherwise processed them.

– virtuosoEventOccurred:forFile:onBearer:withData:

Reports that a tracked event was logged.

- (void)virtuosoEventOccurred:(kVL_LogEvent)event forFile:(nullable NSString *)fileID onBearer:(kVL_BearerType)bearer withData:(long long)data

Parameters

event

The type of event
fileID

The UUID of the relevant asset. Nil for events that don’t relate to a specific asset.
bearer

The network type relating to the event. kVL_Invalid for events that don’t relate to the network state.
data

A numeric value semantically unique to the specific event. Will be 0 when not required. Example: For the download-paused event, contains total bytes downloaded so far for this asset.

– virtuosoDebugEventOccurred:atLevel:

Reports debug data.

- (void)virtuosoDebugEventOccurred:(nonnull NSString *)data atLevel:(kVL_LoggingLevel)level

Parameters

data

A string containing the debug event data
level

The severity level of the log item

Discussion

For non-debug builds and when verbose console option is off, Virtuoso won’t send debug output to the debugging console. Use this method to access log debug output.
