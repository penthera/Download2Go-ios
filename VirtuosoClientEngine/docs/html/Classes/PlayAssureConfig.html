Overview

Provides configuration options for PlayAssure playback.

  headroom

Headroom value, expressed in MB. Default value is 1024 MB (1 GB).

@property (nonatomic, assign) NSInteger headroom

Discussion

Headroom is the amount of free space a PlayAssure session must preserve on disk, expressed in MB. For instance, if headroom is 1GB and the device has 1.2GB free disk space, PlayAssure will begin playback of a 150MB asset, but it will NOT begin playback of a 500MB asset.

  cacheHigherPlayerRequests

If the player receives a request for a video segment at a higher quality than one currently on disk, this flag determines whether to persist the higher quality segment to disk and remove the lower quality segment. If not set (default), the segment request is passed through to the player with an in-memory buffer and its contents are not saved to disk. If no segment exists on disk, it will always be persisted, regardless of this setting.

@property (nonatomic, assign) Boolean cacheHigherPlayerRequests

  targetBitrate

Target bitrate PlayAssure should try and maintain.

@property (nonatomic, readonly) long long targetBitrate

Discussion

A value of - 1 will let the player the choose the the value automatically. 0 will select and lowest bitrate. Any other value PlayAssure will pick the closest available bitrate.
